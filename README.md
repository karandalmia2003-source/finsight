# FinSight -- AI Financial Document Analyzer

Upload financial PDFs (10-K, earnings release, P&L, investor update) and get a structured 5-section brief plus a live dashboard, generated by Claude. Sign up for an account to keep your documents, chat history, and comparisons saved across sessions and devices. Chat with your documents -- including cross-referencing any subset of them against any other subset -- compare any number of them side by side, stress-test assumptions, and see how reported figures stacked up against analyst consensus, all grounded in your documents and live web search.

## Stack

Next.js 14 (App Router), Anthropic SDK (`claude-sonnet-4-6`), Recharts, Tailwind CSS, Postgres (via Vercel's Neon integration) for accounts and session persistence, `bcryptjs` for password hashing, `jose` for signed JWT session cookies.

## Accounts & persistence

FinSight requires a free account. Sign up with an email and password on `/signup`; passwords are hashed with bcrypt before storage. Logging in sets an httpOnly `finsight_session` JWT cookie, and `middleware.js` redirects unauthenticated requests away from the app to `/login`. Every document upload, chat message, stress test, and comparison is persisted server-side in Postgres, scoped to your account -- so your data follows you across browsers and devices instead of living only in one browser's local storage. Use the "Sign out" control at the bottom of the sidebar to end your session.

## Local setup

```bash
npm install
cp .env.example .env.local
# edit .env.local and set:
#   ANTHROPIC_API_KEY=sk-ant-...
#   POSTGRES_URL=postgres://...   (a Postgres connection string, e.g. from Neon)
#   JWT_SECRET=<a long random string>
npm run dev
```

Open http://localhost:3000. You'll be redirected to `/signup` to create an account before you can use the app.

**Important -- enable web search:** Several features below (brief generation, chat, the stress-tester, and the earnings surprise detector) call Claude with the `web_search` tool enabled. Web search must be turned on for your organization in the [Claude Console](https://console.anthropic.com) under Settings -> Privacy before these calls will work. If it's off, those specific Claude calls will return an error -- extraction (PDF -> structured JSON) and cross-document comparison don't use web search and are unaffected either way.

## How it works

1. **Sign up / log in.** `/api/auth/signup` and `/api/auth/login` create accounts and verify credentials against bcrypt hashes, then issue a signed JWT in an httpOnly cookie. `/api/auth/me` reports the current session's email to the UI; `/api/auth/logout` clears the cookie.
2. 2. **Upload & brief.** PDF upload -> sent to `/api/analyze`, which sends the raw PDF to Claude to extract structured financial JSON (revenue, margins, expenses, risks, trends) -- extraction stays grounded purely in the document, no web search. A second Claude call turns that JSON into the 5-section brief, optionally folding in one piece of current market context found via web search. The resulting document record is saved to Postgres under your account.
   3. 3. **Dashboard.** Metric cards, revenue/profit/margin/YoY charts, expense pie, and risk gauge are built entirely from the extracted JSON -- no hardcoded data.
      4. 4. **Chat, with document scoping.** The chat panel (`/api/chat`) sends the *selected* documents' extracted data + briefs to Claude as context, plus a web_search tool, so answers synthesize both your documents and current information. When you have 2+ documents uploaded, a "Scope" row of chips appears above the chat input -- toggle any combination on or off to cross-reference exactly the documents you want (e.g. compare two specific quarters while excluding a third). Your documents' own figures always take precedence over anything found on the web.
         5. 5. **Compare any number of documents.** Once 2+ documents are uploaded, a "Compare Documents" button appears in the sidebar. Select any number of them, in any order, and FinSight computes exact deltas for revenue, gross margin, opex, net income, and YoY growth across all of them in plain JS (no LLM arithmetic involved), then asks Claude for a one-line "what drove this" explanation per metric, grounded in the selected documents themselves.
            6. 6. **Stress Test.** Below the brief, type a "what if" assumption (e.g. "what if gross margin drops 300bps") and Claude reasons through the bottom-line impact using the document's real figures, returning a restated assumption, affected metrics, estimated impact, and a materiality judgment.
               7. 7. **Earnings Surprise.** Right after a document finishes processing, FinSight automatically makes an additional Claude call with web search to find analyst consensus estimates for that company/period, and compares them against the reported figures in a beat/miss/in-line table with a market-context note. Re-runnable any time from its panel.
                  8. 8. **Source priorities.** Wherever web search is used, the system prompts instruct Claude to prioritize Reuters, The Wall Street Journal, the Financial Times, Seeking Alpha, and SEC EDGAR over other sources.
                     9. 9. **Persistence.** Documents, chat history, stress-test results (stored per-document), and comparison results (stored as an account-wide list) are all saved to Postgres, scoped to your signed-in account, so a refresh, browser switch, or new device doesn't lose your session.
                       
                        10. ## Deploying to Vercel
                       
                        11. ```bash
                            vercel
                            ```

                            In the Vercel project settings (Project -> Settings -> Environment Variables), set:
                            - `ANTHROPIC_API_KEY`
                            - - `JWT_SECRET` (a long random string used to sign session cookies)
                             
                              - Then add Vercel's Postgres (Neon) integration to the project (Storage tab -> Connect a Database) -- this automatically provisions the `POSTGRES_URL` and related env vars used by `@vercel/postgres`. Confirm web search is enabled for your organization in the Claude Console (see above) before deploying.
                             
                              - **Note on file size:** Vercel's Hobby plan caps serverless function request bodies at ~4.5MB. Since PDFs are sent as base64 (~33% larger than the raw file), uploads are practically capped around 3MB on Hobby. For larger PDFs, upgrade to a Pro plan or add client-side chunking/compression.

                                middleware.js                   - redirects unauthenticated requests to /login
                                components/

                                  auth.js                    - password hashing, JWT issuing/verification helpers
