# FinSight — AI Financial Document Analyzer

Upload financial PDFs (10-K, earnings release, P&L, investor update) and get a structured 5-section brief plus a live dashboard, generated by Claude. Chat with your documents, compare them against each other, stress-test assumptions, and see how reported figures stacked up against analyst consensus — all grounded in your documents and, now, live web search.

## Stack

Next.js 14 (App Router), Anthropic SDK (`claude-sonnet-4-6`), Recharts, Tailwind CSS, IndexedDB for session persistence.

## Local setup

```bash
npm install
cp .env.example .env.local
# edit .env.local and set ANTHROPIC_API_KEY=sk-ant-...
npm run dev
```

Open http://localhost:3000.

**Important — enable web search:** Several features below (brief generation, chat, the stress-tester, and the earnings surprise detector) call Claude with the `web_search` tool enabled. Web search must be turned on for your organization in the [Claude Console](https://console.anthropic.com) under Settings → Privacy before these calls will work. If it's off, those specific Claude calls will return an error — extraction (PDF → structured JSON) and cross-document comparison don't use web search and are unaffected either way.

## How it works

1. **Upload & brief.** PDF upload → sent to `/api/analyze`, which sends the raw PDF to Claude to extract structured financial JSON (revenue, margins, expenses, risks, trends) — extraction stays grounded purely in the document, no web search. A second Claude call turns that JSON into the 5-section brief, optionally folding in one piece of current market context found via web search.
2. **Dashboard.** Metric cards, revenue/profit/margin/YoY charts, expense pie, and risk gauge are built entirely from the extracted JSON — no hardcoded data.
3. **Chat.** The chat panel (`/api/chat`) sends all uploaded documents' extracted data + briefs to Claude as context, plus a web_search tool, so answers synthesize both your documents and current information — your documents' own figures always take precedence over anything found on the web.
4. **Compare.** Once 2+ documents are uploaded, a "Compare Documents" button appears in the sidebar. Pick any two and FinSight computes exact deltas for revenue, gross margin, opex, net income, and YoY growth in plain JS (no LLM arithmetic involved), then asks Claude for a one-line "what drove this" explanation per metric, grounded in the two documents themselves.
5. **Stress Test.** Below the brief, type a "what if" assumption (e.g. "what if gross margin drops 300bps") and Claude reasons through the bottom-line impact using the document's real figures, returning a restated assumption, affected metrics, estimated impact, and a materiality judgment.
6. **Earnings Surprise.** Right after a document finishes processing, FinSight automatically makes an additional Claude call with web search to find analyst consensus estimates for that company/period, and compares them against the reported figures in a beat/miss/in-line table with a market-context note. Re-runnable any time from its panel.
7. **Source priorities.** Wherever web search is used, the system prompts instruct Claude to prioritize Reuters, The Wall Street Journal, the Financial Times, Seeking Alpha, and SEC EDGAR over other sources.
8. **Persistence.** Documents, chat history, stress-test results (stored per-document), and comparison results (stored as a session-wide list) are all saved to IndexedDB in the browser, so a refresh doesn't lose your session.

## Deploying to Vercel

```bash
vercel
```

Set the `ANTHROPIC_API_KEY` environment variable in the Vercel project settings (Project → Settings → Environment Variables) before deploying, and confirm web search is enabled for your organization in the Claude Console (see above).

**Note on file size:** Vercel's Hobby plan caps serverless function request bodies at ~4.5MB. Since PDFs are sent as base64 (~33% larger than the raw file), uploads are practically capped around 3MB on Hobby. For larger PDFs, upgrade to a Pro plan or add client-side chunking/compression.

## Project structure

```
app/
  api/analyze/route.js       - PDF -> extracted JSON -> brief
  api/chat/route.js          - chat grounded in session documents + web search
  api/compare/route.js       - cross-document comparison commentary
  api/stress-test/route.js   - assumption stress-tester
  api/surprise/route.js      - earnings-surprise vs. analyst consensus
  page.js                    - main app shell + state
components/
  Sidebar.js, Dashboard.js, Brief.js, ChatPanel.js
  CompareModal.js, StressTest.js, SurpriseTable.js
  charts/                    - Recharts components + risk gauge
lib/
  claude.js                  - Anthropic SDK calls (incl. web_search tool wiring)
  prompts.js                 - extraction/brief/chat/compare/stress/surprise system prompts
  compare.js                 - deterministic metric-delta computation
  storage.js                 - IndexedDB persistence
  aggregate.js, format.js, colors.js
```
